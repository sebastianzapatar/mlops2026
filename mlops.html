Definición

¿Qué es MLOps?

MLOps es la disciplina que permite entregar, operar, monitorear y mejorar modelos de ML mediante flujos repetibles y automatizados.

Combina buenas prácticas de ingeniería de software, DevOps, ingeniería de datos y ciencia de datos.

Componentes

Partes fundamentales de MLOps

Feature Store

Centraliza variables usadas para entrenamiento e inferencia, evitando diferencias entre offline y online.

Versionado de datos

Permite reproducir experimentos y auditar cómo cambian los datasets.

Metadata Store

Guarda parámetros, métricas, artefactos, seeds, ambiente y resultados.

Model Registry

Organiza modelos, versiones, etapas, aprobaciones y transición a producción.

Serving

Expone el modelo mediante APIs, batch jobs, streaming o servicios embebidos.

Monitoring

Observa calidad, latencia, drift, sesgo, errores y uso real del modelo.

API

¿Qué es una API en ML?

Una API es una puerta controlada para que otras aplicaciones usen un modelo sin conocer su implementación interna.

  • Recibe datos de entrada.
  • Valida el formato.
  • Ejecuta el modelo.
  • Retorna una predicción.
POST /predict { "edad": 35, "ingresos": 4200000, "historial": "bueno" } 200 OK { "riesgo": "bajo", "probabilidad": 0.82 }

Tracking con MLflow

import mlflow from sklearn.metrics import accuracy_score with mlflow.start_run(): mlflow.log_param("modelo", "RandomForest") mlflow.log_param("n_estimators", 100) model.fit(X_train, y_train) preds = model.predict(X_test) mlflow.log_metric("accuracy", accuracy_score(y_test, preds)) mlflow.sklearn.log_model(model, "model")
Calidad y monitoreo

¿Qué se debe monitorear?

Técnico

Latencia, errores, disponibilidad, consumo de CPU/memoria.

Datos

Cambios en distribución, valores faltantes, outliers y drift.

Modelo

Precisión, recall, F1, calibración, sesgo y degradación.

Sin monitoreo, un modelo puede seguir respondiendo técnicamente bien mientras sus predicciones pierden valor para el negocio.
MLOps + GenAI

Actualización 2026: de MLOps a LLMOps

MLOps tradicional

Modelos predictivos, datasets estructurados, métricas supervisadas, pipelines de entrenamiento y serving.

LLMOps / GenAI Ops

Prompts, RAG, evaluación de respuestas, trazas, costos por token, seguridad, guardrails y observabilidad de agentes.

La idea central es la misma: llevar IA a producción con control, medición, trazabilidad y mejora continua.

Cierre

Ideas clave

  • MLOps no es una herramienta: es una práctica de ingeniería.
  • El ciclo no termina al entrenar; empieza cuando el modelo se usa.
  • Versionar datos, código y modelos permite reproducibilidad.
  • MLflow es una excelente puerta de entrada para experimentar con MLOps.
  • APIs permiten integrar modelos con aplicaciones reales.
  • Monitorear y reentrenar evita que el modelo se vuelva obsoleto.
