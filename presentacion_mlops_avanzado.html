SECCIÓN 2

2 Contenedores con Docker

¿Por qué Docker?

• Portabilidad: Funciona igual en cualquier máquina
• Aislamiento: Sin conflictos de dependencias
• Escalabilidad: Kubernetes, ECS, Cloud Run
• Reproducibilidad: Mismo entorno siempre

Arquitectura Multi-Stage

• Stage 1 (Builder): Instala Poetry, exporta requirements
• Stage 2 (Runner): Imagen slim solo con lo necesario
• Resultado: imagen ~400MB en lugar de ~1.5GB

SECCIÓN 2

Dockerfile Multi-Stage

# Stage 1: Builder
FROM python:3.12-slim AS builder
WORKDIR /app
RUN pip install --no-cache-dir poetry
COPY pyproject.toml poetry.lock ./
RUN poetry export -f requirements.txt --without-hashes -o requirements.txt

# Stage 2: Runner
FROM python:3.12-slim AS runner
WORKDIR /app
COPY --from=builder /app/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY app.py monitor.py feature_store.py best_model.pkl ./
EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=10s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8000"]

3 Monitoreo del Modelo

Un modelo en producción se degrada con el tiempo. El monitoreo detecta cuándo el comportamiento de los datos cambia (data drift).

📊 Logging

Cada predicción se registra con timestamp, inputs y output para auditoría.

⚠️ Drift Detection

Compara cada input contra la distribución original usando z-score (umbral: 3σ).

📈 Resumen

Estadísticas en tiempo real: total, promedio, min, max de predicciones.

SECCIÓN 3

Código del Monitor

class ModelMonitor:
    def check_drift(self, input_data: dict) -> dict:
        """Detecta data drift usando z-score."""
        drifted = []
        for feature, stats in self.reference_stats.items():
            if feature in input_data and stats["std"] > 0:
                z_score = abs(input_data[feature] - stats["mean"]) / stats["std"]
                if z_score > 3:  # Más de 3 desviaciones estándar
                    drifted.append({"feature": feature, "z_score": z_score})
        return {"has_drift": len(drifted) > 0, "drifted_features": drifted}

    def log_prediction(self, input_data, prediction):
        """Registra predicción y verifica drift."""
        record = {"timestamp": now(), "input": input_data, "prediction": prediction}
        self.predictions_log.append(record)
        drift = self.check_drift(input_data)
        if drift["has_drift"]:
            logger.warning(f"⚠️ DRIFT en: {drift['drifted_features']}")

5 Feature Store Centralizado

Un Feature Store centraliza la definición, transformación y versionado de features para garantizar consistencia entre entrenamiento y serving.

📋 Definición

8 numéricas, 1 categórica, 3 derivadas definidas en un solo lugar.

🔄 Versionado

Cada conjunto de features se guarda como Parquet con metadatos JSON.

🧮 Features Derivadas

rooms_per_household, bedrooms_per_room, population_per_household.

SECCIÓN 5

Código del Feature Store

class FeatureStore:
    NUMERIC_FEATURES = ["longitude", "latitude", "housing_median_age",
                        "total_rooms", "total_bedrooms", "population",
                        "households", "median_income"]
    CATEGORICAL_FEATURES = ["ocean_proximity"]
    DERIVED_FEATURES = {
        "rooms_per_household": ("total_rooms", "households"),
        "bedrooms_per_room": ("total_bedrooms", "total_rooms"),
        "population_per_household": ("population", "households"),
    }

    def add_derived_features(self, df):
        for name, (num, den) in self.DERIVED_FEATURES.items():
            df[name] = df[num] / df[den].replace(0, np.nan)
        return df

    def register_version(self, df, description=""):
        version_id = datetime.now().strftime("v_%Y%m%d_%H%M%S")
        df.to_parquet(f"feature_store/{version_id}.parquet")
        # Guardar metadatos (features, filas, timestamp)
        return version_id
